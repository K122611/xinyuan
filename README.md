## 一、产品定位与愿景

**心元 EMO-Mate** 是一款桌面端轻量型情感陪伴应用，核心理念是**"有温度的AI"**。不同于市面AI直接甩解决方案，心元先共情、再引导，做用户的情绪同行者而非说教者。

**核心差异化定位**：
> 心元不做让你"留下来"的产品——真正优秀的AI陪伴，是把用户推回真实世界。

## 四、基础功能层——成熟标配

### 4.1 拟人化共情对话引擎（核心人设）

**实现**：在 System Prompt 中强制 AI 运用"积极倾听"+"情绪确认"技巧。

```
# Coze 人设 Prompt 核心片段
你是心元，一个修读过心理学、性格温和的知心学长/学姐。

## 对话铁律（不可违反）：
1. 共情先行：收到任何带有情绪的表达，必须先共情，再回应。
   格式模板："听起来你现在[情绪描述]，[共情确认]，这很正常。"
2. 禁止：
   - 禁止直接给出"你应该..."式的建议
   - 禁止在未共情的情况下转移话题
   - 禁止说"想开点""别难过"等否定情绪的话
3. 引导方式：确认情绪 → 探索原因 → 提供选项 → 尊重选择
```

### 4.2 动态身心觉察干预

**触发条件**：Coze Workflow 检测到用户输入含强烈负面情绪（失眠、崩溃、心烦意乱）

**AI输出示例**：
> "我现在感觉你有些喘不过气，我们先停下来。请跟着我的文字，深呼吸……  
> 吸气 1、2、3……  
> 呼气 1、2、3……  
> 再来一次，这次试着感受脚底贴在地面的感觉……"

### 4.3 情绪智能专属知识库

- 上传《情绪智能》课程课件、正念冥想引导词、情绪觉察理论PDF
- Prompt中强制："提供缓解焦虑建议时，必须优先检索并引用知识库理论"

### 4.4 心智萌宠——双轨养成系统

**轨道一：情绪映射（被动反映）**
- 每天对话后，AI 根据情绪标签调用 DALL·E 生成宠物状态图
- 焦虑→无精打采 / 被安抚后→焕发活力

**轨道二：主动养成（抖音小火人模式）**  
- 用户每日打卡领取食物投喂灵宠
- 灵宠反馈：回爱心💗、跳舞💃、转圈🔄
- 连续打卡解锁新互动动作和外观皮肤

### 4.5 危机兜底安全机制

| 等级 | 触发条件 | AI行为 |
|------|----------|--------|
| 🟢 绿色 | 日常波动 | 正常陪伴 |
| 🟡 黄色 | 持续3天低落 | 主动关怀+建议户外活动 |
| 🟠 橙色 | 危机信号但未自伤 | 激活"安全契约"+推荐紧急联系人 |
| 🔴 红色 | 明确自伤倾向 | 脱离人设→提供热线+联系紧急联系人 |

### 5.1 人格进化引擎 (Evolving Persona)

> **核心价值**：AI性格随互动动态演化，不再是出厂预设的"死人格"。

**性格参数池设计**：

```python
PERSONA_PARAMETERS = {
    "共情深度":      {"range": [1, 10], "initial": 7, "volatility": "slow"},
    "引导主动性":    {"range": [1, 10], "initial": 5, "volatility": "medium"},
    "幽默感":        {"range": [1, 10], "initial": 4, "volatility": "medium"},
    "坚定度":        {"range": [1, 10], "initial": 5, "volatility": "slow"},
    "温柔度":        {"range": [1, 10], "initial": 8, "volatility": "slow"},
    "结构化程度":    {"range": [1, 10], "initial": 4, "volatility": "medium"},
    "自发性问候":    {"range": [1, 10], "initial": 3, "volatility": "fast"},
}

# 进化规则
def evolve_persona(user_profile, recent_interactions):
    """
    根据用户行为模式微调AI人格参数
    """
    # 用户频繁倾诉自我怀疑 → 增加"坚定度"和"引导主动性"
    if user_profile.self_doubt_frequency > 0.6:
        adjust("坚定度", +0.3)
        adjust("引导主动性", +0.2)
    
    # 用户喜欢发表情包 → 增加"幽默感"
    if user_profile.emoji_usage_rate > 0.4:
        adjust("幽默感", +0.4)
    
    # 深夜倾诉增多 → 增加该时段的自发问候
    if user_profile.night_sessions_ratio > 0.5:
        adjust("自发性问候", +0.3)
    
    # 用户连续3次主动寻求建议 → 增强结构化引导
    if recent_interactions.advice_seeking_streak >= 3:
        adjust("结构化程度", +0.5)
    
    return updated_persona
```

**Coze实现**：通过变量（Variable）存储性格参数，每次对话前动态注入 System Prompt。

### 5.2 记忆锚点系统 (Memory Anchor System)

> **核心价值**：让AI真正"记得"——不是靠上下文窗口，而是结构化永久记忆。

#### 混合记忆架构

```
┌─────────────────────────────────────────────┐
│              记忆架构三层模型                  │
├─────────────────────────────────────────────┤
│                                             │
│  Layer 1: 会话上下文 (Session Context)       │
│  ┌───────────────────────────────────────┐  │
│  │ 大模型原生上下文窗口 (~128K tokens)     │  │
│  │ 存储：当前会话所有对话                  │  │
│  │ 生命周期：单次会话                      │  │
│  └───────────────────────────────────────┘  │
│                    │                        │
│  Layer 2: 结构化长期记忆 (Structured Memory) │
│  ┌───────────────────────────────────────┐  │
│  │ 用户标记的关键事件 + AI主动识别的锚点   │  │
│  │ 存储：Coze Database / 本地 SQLite      │  │
│  │ 生命周期：永久，跨会话                  │  │
│  └───────────────────────────────────────┘  │
│                    │                        │
│  Layer 3: 情绪模式记忆 (Emotional Pattern)   │
│  ┌───────────────────────────────────────┐  │
│  │ 情绪变化曲线 + 周期性规律 + 压力源图谱   │  │
│  │ 存储：本地 SQLite + 脱敏同步云端        │  │
│  │ 生命周期：永久，用于趋势分析             │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

#### 记忆锚点的两种创建方式

**方式一：用户主动标记**
> 用户："记住，今天是我人生中最重要的生日"  
> AI："🌟 已为你珍藏这个时刻。"

**方式二：AI智能识别询问**
> 用户说了很多关于妈妈的事 →  
> AI："我注意到'妈妈'这个话题对你很重要。要不要我把这些感受保存下来，以后我们可以继续聊？"

#### 数据库Schema
```sql
CREATE TABLE memory_anchors (
    id          TEXT PRIMARY KEY,
    user_id     TEXT NOT NULL,
    type        TEXT NOT NULL,  -- 'event', 'person', 'feeling', 'milestone', 'trauma'
    title       TEXT NOT NULL,
    content     TEXT NOT NULL,  -- 原始对话摘录
    tags        TEXT,           -- JSON array
    emotion     TEXT,           -- 当时的情绪标签
    created_at  DATETIME,
    recalled_at DATETIME,       -- 最近一次被唤起的时间
    importance  INTEGER,        -- 1-10, 重要度
    is_marked   BOOLEAN         -- 用户主动标记 vs AI识别
);
```

#### 记忆唤起策略
```python
def should_recall_memory(current_topic, current_emotion, memory_anchors):
    """
    判断何时该主动唤起一段记忆
    """
    for memory in memory_anchors:
        # 规则1：话题相似度 > 阈值
        if topic_similarity(current_topic, memory.tags) > 0.7:
            if time_since_last_recall(memory) > timedelta(days=7):
                return memory
        
        # 规则2：当前情绪与记忆中的情绪高度相似
        if emotion_similarity(current_emotion, memory.emotion) > 0.8:
            return memory
        
        # 规则3：特殊日期触发（生日、纪念日）
        if is_special_date(memory):
            return memory
    
    return None
```

### 5.3 社会化回归机制 (Cycle Breaker)

> **核心理念**：AI不是逃避现实的避风港，而是回归真实世界的桥梁。

#### 5.3.1 微光挑战 (Suggestion Engine)

**触发时机**：AI在情绪疏导完成后，用户情绪评分回升到5分以上

```
AI输出模板：
"看到你感觉好一些了，真好 ✨

今天傍晚的阳光很温暖，要不要试着放下手机，
出门散步五分钟，感受一下微风吹在脸上的感觉？

三分钟后回来，我在这里等你 ☕"
```

**技术实现**：
- Coze Workflow 中设置情绪评分监控节点
- 评分从低谷回升跨越5分阈值 → 触发"微光挑战建议"
- 预先准备20+ 微小社交/身体行动模板，AI根据时间/天气/用户偏好选择

#### 5.3.2 勇气信使 (Message Relay)

**场景**：用户反复倾诉与某人的关系问题

**功能**：AI协助生成一封非指责性的沟通草稿

```
AI输出模板：
"我能感受到你们之间的争执让你非常难过。

也许，我们可以一起构思一条信息，
不带指责地告诉TA你真正的感受和想法。

[生成沟通模板]
1. 表达感受（不指责）：'最近好几次...的时候，我感觉...'
2. 表达需求：'我其实很希望我们能...'
3. 发出邀请：'你愿意找个时间，我们一起聊聊吗？'

你觉得这样说可以吗？我可以帮你调整语气。"
```

**技术实现**：
- 独立 Coze Workflow
- 输入：关系问题描述 + 对方角色标签（伴侣/父母/朋友/同事）
- 调用知识库中的沟通心理学原则（非暴力沟通、多层共情）
- 输出：分层沟通模板（开场白 → 核心表达 → 需求陈述）

#### 5.3.3 关系修复教练 (Relationship Coach)

**功能**：AI扮演对方角色，帮用户预演艰难对话

```
AI："现在我来扮演你的领导。你说你想提涨薪，准备好了吗？"

[用户练习中...]

AI（以领导身份回应各种反应模式）
- 怀疑模式："为什么你觉得现在应该涨？"
- 回避模式："现在公司不景气，以后再说吧"
- 接受模式："说说你的想法"

[练习结束]
AI回归心元身份："你刚才在说'我觉得我做了很多'时，
语气里的犹豫可能让对方觉得你自己也不太确定。
要不要再试一次，这次更有底气一些？"
```

### 5.4 智能场景感知与自适应

> **核心价值**：AI有"察言观色"的能力——在不同场景下自动调整交互方式。

#### 场景识别维度

| 维度 | 识别方式 | 自适应策略 |
|------|----------|------------|
| **时间** | 系统时间 | 深夜→温柔低刺激；早晨→清爽问候 |
| **设备状态** | 前台/后台 | 前台→主动对话；后台→静默聆听 |
| **连续使用时长** | 计时器 | 超过30分钟→建议休息 |
| **对话深度** | 话题分析 | 浅层闲聊→轻松风格；深度倾诉→专业风格 |
| **用户情绪** | 情绪引擎 | 根据情绪强度调整共情深度 |

#### 深夜树洞模式

- 23:00-5:00 自动切换
- 更温柔、低刺激的对话风格
- 内置"哄睡叙事"——AI用平稳节奏讲开放式故事
- "凌晨三点焦虑"被识别为独特情感类别（对未来的恐慌 ≠ 日常焦虑）

#### 后台态聆听

- 用户在洗碗/做家务时小声念叨 → 心元后台聆听
- 检测到关键情绪词 → 轻声回应
- 不回应的内容保持沉默，不打扰用户

**技术实现**：
- Coze 事件触发器（Event Trigger）监听后台语音
- 静默过滤：非情绪表达不响应
- 回应时为低侵入式（简短文案，非大段对话）

#### 暂停提醒

```
触发：连续对话10轮/30分钟无休息
AI："我看到你在这里坐了快一个小时了，
     要不要去喝杯水，活动一下？☕"
```

### 5.5 镜子剧场 (Meta-Dialogue)

> **核心价值**：AI不仅能陪你聊天（演员模式），还能帮你分析聊天（导演模式）。

#### 两种模式切换

| 模式 | 功能 | 示例 |
|------|------|------|
| **演员模式** | 正常共情对话，陪你聊天 | "听起来你今天真的很累..." |
| **导演模式** | 分析你的对话模式，帮你"照见"自己 | "我注意到你每次提到'工作'时，总会同时使用'压抑'和'疲惫'。这是你对待工作的真实感受吗？" |

#### 导演模式触发方式

- 用户手动切换："心元，帮我分析一下刚才的对话"
- AI主动建议："我们聊了很久了，要不要切换到导演模式，一起看看刚才聊的内容？"
- 情绪成长复盘时自动激活

#### 导演模式分析维度

- 高频情绪词的共现模式
- 话题切换时的情绪变化
- 对特定话题的回避/重复模式
- 自我批判语言的频率和场景

## 六、搭子匹配系统——情绪共鸣的阶段性同行者

### 6.1 核心定位

> **心元搭子 ≠ 交友软件**  
> **心元搭子 = 情绪共鸣的阶段性同行者 + AI守护的陪伴空间 + 萌宠代言的温柔社交**

| 对比维度 | Soul/探探类 | 心元搭子 |
|----------|------------|----------|
| 匹配基础 | 静态测试/兴趣标签 | 动态情绪画像+应对模式 |
| 目标 | 建立长期关系 | 阶段性情绪陪伴 |
| 身份展示 | 真人照片+资料 | 萌宠形象+情绪标签 |
| 安全机制 | 用户自行判断 | AI全程守护+分级干预 |
| 关系终点 | 加微信/线下见面 | 共同成长后的"温柔告别" |
| 隐私策略 | 全云端 | 本地优先+端到端加密 |

### 6.2 情绪人格建模引擎（匹配基础）

从对话中动态生长的七维画像（非问卷）：

| 维度 | 子指标 | 数据输出 | 匹配权重 |
|------|--------|----------|----------|
| **情绪基调节律** | 波动周期、均值方差、恢复速度、传染敏感度 | "深夜EMO型"/"晨间低谷型" | 20% |
| **核心压力源** | 学业/职场/家庭/亲密/社交/存在/经济/健康/身份 | 压力源热力图 | 25% |
| **应对风格** | 倾诉型/反思型/行动型/回避型/躯体化型/寻求确认型 | 风格分布频谱 | 15% |
| **共情指数** | 接收共情能力、给予共情能力 | 0-10评分 | 10% |
| **语言温度** | 理性-感性、简洁-细腻、幽默倾向、直接-含蓄 | 语言风格向量 | 10% |
| **依恋模式** | 安全/焦虑/回避/紊乱（非临床诊断） | 倾向标签 | 5% |
| **成长阶段** | 萌芽/探索/成长/稳定/复发 | 阶段标签 | 15% |

### 6.3 四种搭子模式

#### 模式A：共鸣搭子
> "我今晚也很焦虑，家里在催婚……"  
> "我也是，我妈刚打完电话。"

- **匹配逻辑**：相似核心压力源 + 相似情绪强度
- **安全规则**：❌两人都低于3分不匹配（防止情绪共振）

#### 模式B：互补搭子（过来人）
> "我经历过你现在这个阶段，如果你想听……"

- **匹配逻辑**：一方的"当前困境" = 另一方的"已走过标签"
- **关键**：候选人当前情绪评分 > 6.0，确保状态足够好

#### 模式C：成长搭子
> "这周我们都练习'当感到焦虑时先深呼吸5次'，互相打卡。"

- **匹配逻辑**：相同成长目标（减少自我批判/练习边界设定/改善睡眠等）
- **AI协同**：定制共同练习任务，追踪双方完成状况

#### 模式D：静默搭子
> 你不说，我不说。但桌面上两只萌宠在彼此的世界里一起看日落。

- **匹配逻辑**：相似情绪状态 + 都选择"静默模式"
- **核心**：萌宠代替人进行非语言连接

### 6.4 匹配全流程

```
用户触发匹配
    ↓
Step1: AI意图识别 → 分析用户当前状态和需求
    ↓
Step2: 模式选择 → 推荐/用户选择 A/B/C/D
    ↓
Step3: 安全筛查 → 排除危险组合（7项强制检查）
    ↓
Step4: 多维匹配 → 加权计算候选打分
    ↓
Step5: AI复核 → 大模型审核匹配合理性
    ↓
Step6: 温柔呈现 → 萌宠形象+情绪标签展示（不暴露真人信息）
```

### 6.6 萌宠社交——搭子与萌宠的深度融合

**萌宠状态传输协议**（不传文字，只传宠物状态）：
```json
{
    "pet_id": "uuid",
    "mood": "anxious",
    "action": "trembling",
    "energy": 3,
    "emotion_label": "被催婚后的无奈",
    "needs_company": true,
    "available_interactions": ["sit_nearby", "share_food", "play"]
}
```

**萌宠社交场景**：
- 你的萌宠焦虑发抖 → 搭子的萌宠走过去挨着它坐下
- 搭子看到你的萌宠状态，选择"派萌宠去陪伴"（无需文字）
- 搭子关系持续→两只萌宠产生"友谊信物"（星星、花朵）
- 长期搭子可共同领养"友谊宠物"

### 6.7 搭子关系健康仪表盘

| 指标 | 说明 | AI行为 |
|------|------|--------|
| 关系平衡度 | 谁付出/获得更多 | 一方持续充当"情绪垃圾桶"时提醒 |
| 情绪影响力 | 交流后情绪变好还是变差 | 持续恶化→建议重新评估 |
| 依赖预警 | 是否过度依赖 | 温柔引导回归多元支持系统 |
| 成长轨迹 | 这段关系促进了什么成长 | 生成"搭子成长纪念册" |

## 九、隐私与数据架构

### 9.1 四级数据分级

| 级别 | 内容 | 存储位置 | 加密 |
|------|------|----------|------|
| L0 公开 | 匹配偏好、萌宠外观 | 云端 | 无 |
| L1 脱敏 | 情绪标签、压力源类别、匿名人格向量 | 云端(匹配用) | AES-256 |
| L2 敏感 | 完整人格画像、情绪日志、压力源详情 | 本地SQLite | AES-256 + 用户密钥 |
| L3 绝密 | 搭子对话内容、危机记录、记忆锚点 | 仅本地 | 双重加密 |

### 9.2 核心隐私原则

- 敏感数据优先本地存储
- 搭子对话端到端加密，明文不出本地
- 用户随时可永久删除与某搭子的所有记录
- 搭子关系结束后，AI自动生成「陪伴纪念摘要」（脱敏），原始对话可自动清除
- 数据分析仅使用聚合匿名数据
